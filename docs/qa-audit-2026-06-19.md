## 1. Cambios implementados

### 1.1 Rol `receptionist`

- Migración: [`supabase/migrations/20260619000000_add_receptionist_role.sql`](supabase/migrations/20260619000000_add_receptionist_role.sql)

### 1.2 API de recepción

| Endpoint | Descripción |
|----------|-------------|
| `GET /api/reception/lookup?cedula=` | Estado miembro + membresía + dentro/fuera |
| `POST /api/reception/check-in` | Entrada asistida + audit log |
| `POST /api/reception/check-out` | Salida asistida + audit log |
| `GET /api/stats/reception` | KPIs recepción |

- Normalización cédula: `V-12345678` ↔ `12345678` ([`src/lib/cedulaUtils.ts`](src/lib/cedulaUtils.ts))
- Lógica compartida kiosk/staff: [`src/api/attendance/attendanceCore.ts`](src/api/attendance/attendanceCore.ts)

### 1.3 Permisos recepcionista

| Puede | No puede |
|-------|----------|
| Panel recepción, miembros (solo clientes), pagos (aprobar/rechazar) | Configuración, reportes, auditoría, membresías CRUD |
| Asignar membresía a miembro | Crear admin/trainer, eliminar usuarios |
| Ver quién está dentro / log del día | Rutinas, ejercicios |

### 1.4 UI

- [`src/pages/Reception.tsx`](src/pages/Reception.tsx) — panel principal
- [`src/pages/reception/ReceptionDashboard.tsx`](src/pages/reception/ReceptionDashboard.tsx) — dashboard
- Nav recepcionista: Dashboard, Recepción, Miembros, Pagos, Perfil

### 1.5 Aislamiento por rol

- Trainer: listado de miembros filtrado por rutinas asignadas (API)
- Ruta `/history` solo para `member` (trainer ya no accede por error)
- Código muerto de trainer en `ActiveWorkout` eliminado
- Cédula obligatoria en registro staff

### 1.6 Responsive (cliente / trainer)

- Hook [`src/hooks/useIsMobile.ts`](src/hooks/useIsMobile.ts) (767px)
- Dashboard cliente: CTA fijo “Empezar entrenamiento” en móvil
- QuickAction: touch targets 72px mínimo
- MemberRoutine / Exercises: botones e inputs 48px+
